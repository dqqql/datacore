## 结论区

## 实现目标
- 本次前端范围：
  - 登录页
  - 首角色创建引导
  - 玩家端仪表盘、角色页、背包页、公共商店页、玩家交易页
  - 管理员端用户管理、商店管理、密码池管理、审计页
- 主流程：
  - 登录 -> 强制创建首个角色 -> 进入仪表盘 -> 选择当前角色 -> 查看背包与数值 -> 公共商店购买 / 玩家交易上架与购买

## 页面/组件拆分

- `AppShell`：全局导航、用户信息、当前角色切换入口
- `LoginForm`：账号密码登录表单
- `FirstCharacterWizard`：首次登录创建首角色
- `DashboardSummary`：荣誉值、角色摘要、快捷入口
- `CharacterSwitcher`：全局当前角色切换
- `CharacterEconomyPanel`：展示与编辑当前角色金币、声望
- `InventoryTable`：角色背包列表、物品类型、价格、数量
- `PrivateItemForm`：新增私人物品
- `ShopTabs`：公会商店 / 荣誉商店 / 官方规则书入口切换
- `ShopItemList`：商店商品列表、分类筛选、购买按钮
- `SellbackPanel`：公共物品回收入口
- `MarketListingTable`：玩家交易挂单列表
- `CreateListingAction`：从私人物品创建挂单
- `AdminUserTable`：管理员账号与普通用户管理
- `HonorAdjustForm`：荣誉值发放/扣减
- `AdminShopEditor`：商店商品维护
- `OneTimePasswordPanel`：一次性密码池刷新与查看
- `AuditLogTable`：经济修改和后台操作记录

## 核心状态

- 数据加载：
  - 当前用户信息
  - 当前选中角色
  - 角色列表
  - 当前角色背包
  - 商店商品列表
  - 玩家交易挂单列表
  - 管理员后台表格数据

- 空状态：
  - 无角色：强制进入首角色创建
  - 背包为空：引导添加私人物品或前往商店
  - 市场无挂单：显示暂无在售物品
  - 商店无商品：显示空列表和管理员提示

- 错误状态：
  - 登录失败
  - 购买失败（余额不足、库存不足、物品已售）
  - 上架失败（物品不可交易）
  - 一次性密码校验失败

- 提交反馈：
  - 表单提交后给出明确成功/失败反馈
  - 购买、回收、上架、下架成功后局部刷新列表
  - 对影响余额和库存的操作优先展示最新数值

## 风险与注意
- 风险点：
  - 如果“当前选中角色”处理不统一，多个页面会反复让用户选角色
  - 背包、市场、商店三处数据刷新时机不一致，容易让用户以为操作失败
  - 管理员与玩家权限混在同一页面，会增加误操作风险

- 简化方式：
  - 首版用全局当前角色状态
  - 列表页优先用分页或基础表格，不引入复杂状态库
  - 前端数据获取尽量围绕页面级请求和局部刷新，不做全局缓存体系

### 不做的事项及原因
- 不做复杂主题切换和视觉包装：当前重点是主链路跑通
- 不做市场拍卖、议价类页面：与已确认范围冲突
- 不做小游戏入口的正式交互：只保留导航占位或后续扩展位
