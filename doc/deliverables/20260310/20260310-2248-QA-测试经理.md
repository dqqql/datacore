## 结论区

### 测试目标
- 当前版本核心风险：
  - 角色、背包、商店、市场、后台审计虽然都已接入真实逻辑，但跨页面数据联动较多，最容易在“结算正确性”和“状态刷新一致性”上出问题。
  - 玩家交易与公共商店都依赖事务，若流程判断有遗漏，容易出现“钱扣了但物品没变”或“物品变了但日志没记”的问题。
- 本轮测试重点：
  - 用最少手工动作覆盖首版 MVP 主链路
  - 优先验证金额、物品归属、角色归属、审计记录是否一致
  - 在正式测试清单之前，先形成一套用户验收可直接执行的检查步骤

### MVP 范围界定
- 本轮测试优先主流程与高风险异常
- 不要求完整自动化
- 不要求性能压测、安全专项、复杂兼容性矩阵

## 必测清单

### 用例 1：管理员登录并创建普通账号
- 目标：确认管理员入口、账号创建、中文账号基本可用
- 步骤：
  1. 使用管理员账号登录
  2. 进入“账号与荣誉管理”
  3. 创建至少 2 个普通玩家账号
- 预期结果：
  - 登录成功
  - 新账号出现在账号列表
  - 账号名与显示名一致

### 用例 2：玩家首角色创建与当前角色切换
- 目标：确认首角色强制引导和当前角色全局状态
- 步骤：
  1. 使用玩家 A 登录
  2. 首次进入时完成首角色创建
  3. 在角色页新增第 2 个角色
  4. 切换当前角色并进入角色详情
- 预期结果：
  - 无角色时被强制引导创建首角色
  - 新角色创建成功
  - 当前角色切换后详情页展示正确角色数据

### 用例 3：金币/声望修改与审计留痕
- 目标：确认角色经济数值可编辑且有记录
- 步骤：
  1. 在角色详情页修改金币
  2. 再修改声望
  3. 管理员进入审计日志页查看最新记录
- 预期结果：
  - 数值保存成功
  - 审计日志出现 `CHARACTER_GOLD_UPDATED`、`CHARACTER_REPUTATION_UPDATED`
  - 改前改后值可追溯

### 用例 4：私人物品录入
- 目标：确认玩家能把私人物品写入背包
- 步骤：
  1. 在角色详情页录入私人物品
  2. 填写名称、描述、数量、单价
  3. 保存后查看背包表格和审计日志
- 预期结果：
  - 背包新增对应私人物品
  - 状态为“背包中”
  - 审计日志出现 `PRIVATE_ITEM_CREATED`

### 用例 5：公共商店购买与公共物品半价回收
- 目标：确认公共物品买入、入包、回收、返款全链路正确
- 步骤：
  1. 先保证当前角色金币足够
  2. 在公会商店购买 1 个商品
  3. 回到角色详情页确认背包新增公共物品
  4. 在“公共物品回收”区回收 1 个商品
- 预期结果：
  - 购买时金币正确扣减
  - 公共物品进入当前角色背包
  - 回收后金币按当前售价一半返还
  - 审计日志出现 `SHOP_PURCHASED` 与 `SHOP_SELLBACK`

### 用例 6：荣誉值发放与荣誉商店购买
- 目标：确认账号荣誉值和荣誉商店链路正确
- 步骤：
  1. 管理员为玩家 A 发放荣誉值
  2. 玩家 A 进入荣誉商店购买 1 个商品
  3. 回角色页确认物品入包
  4. 管理员查看审计日志
- 预期结果：
  - 荣誉值调整成功且不能被扣成负数
  - 购买时扣的是账号荣誉值，不是角色金币
  - 物品仍然进入当前角色背包
  - 审计日志出现 `USER_HONOR_UPDATED` 和 `SHOP_PURCHASED`

### 用例 7：玩家交易上架、下架、重新上架
- 目标：确认私人物品市场前置动作正确
- 步骤：
  1. 玩家 A 在角色页选择一个私人物品上架
  2. 进入市场页确认挂单出现
  3. 对自己的挂单执行下架
  4. 回角色页修改单价后重新上架
- 预期结果：
  - 只有私人物品可上架
  - 上架后状态变为“已上架”
  - 下架后可重新上架
  - 审计日志出现 `MARKET_LISTED`、`MARKET_CANCELLED`

### 用例 8：玩家交易自动成交
- 目标：确认买卖双方结算和物品转移都正确
- 步骤：
  1. 玩家 A 上架一个私人物品
  2. 玩家 B 登录并保证当前角色金币足够
  3. 玩家 B 在市场页购买该挂单
  4. 分别查看玩家 A 和玩家 B 的角色详情与管理员审计页
- 预期结果：
  - 玩家 B 金币减少
  - 玩家 A 对应卖方角色金币增加
  - 物品转移到玩家 B 当前角色背包
  - 挂单不再出现在 `ACTIVE` 列表中
  - 审计日志出现 `MARKET_PURCHASED`

### 用例 9：密码池刷新与后台查看
- 目标：确认管理员安全后台最小闭环可用
- 步骤：
  1. 管理员进入“密码池”
  2. 刷新一次密码池
  3. 查看批次状态、数量、预览密码
  4. 再进入审计日志查看记录
- 预期结果：
  - 新批次生成成功
  - 旧批次被标记为失效
  - 审计日志出现 `SHOP_PASSWORD_POOL_REFRESHED`

## 异常与边界

- 异常场景 1：荣誉值扣减超过当前余额
  - 预期：系统阻止提交，不允许出现负数荣誉值

- 异常场景 2：金币不足时购买商店物品
  - 预期：购买失败，背包和金币都不应变化

- 异常场景 3：金币不足时购买市场挂单
  - 预期：购买失败，挂单仍保持 `ACTIVE`

- 异常场景 4：尝试购买自己的市场挂单
  - 预期：系统阻止，不允许自买自卖

- 异常场景 5：尝试将公共物品上架市场
  - 预期：角色页不提供上架动作，或后端拒绝请求
