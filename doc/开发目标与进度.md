成功标准：
- 玩家可用指定角色完成公共商店购买

## MVP 范围

### 必须做（当前版本）
- [x] 账号登录与管理员创建用户
- [x] 角色、背包、金币/声望基础管理
- [x] 荣誉值管理
- [x] 公共商店购买与半价回收
- [x] 玩家交易自动成交
- [x] 审计日志与后台补齐

### 可延后（下一版本）
- 炼金、种植、锻造小游戏：当前不影响首版闭环
- 自动化规则：经济系统仍可能调整

### 不做（明确排除）
- 玩家自助注册：首版统一由管理员创建
- 议价、拍卖：会明显增加交易复杂度

1. 管理员密码文件沿用 `./secrets/admin-password.txt`
2. 首版 MVP 主流程已覆盖“私人物品录入 -> 公共商店购买/回收 -> 玩家交易自动成交”

### 验收关注点
1. 管理员创建普通账号与登录是否顺畅

4. 公会商店与荣誉商店购买、公共物品半价回收是否结算正确
5. 荣誉值发放、密码池刷新、审计日志追溯是否可用
6. 以下属于当前已知 MVP 边界，不作为验收阻塞：
